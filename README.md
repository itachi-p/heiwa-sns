# Nagi SNS（仮名）

（ベータテスト状態で2026年4月中の公開予定）

**いいね数の累積や通知、フォロー機能やブロック機能を必要としない設計思想**

- 上記の設計思想をより穴のない状態で実現できるよう、ユーザーからのフィードバック対応も含め継続的に改善予定
- AIによる「投稿単位」の攻撃性判定結果を使った表示優先度の減衰（※各ユーザーごとの設定により、いつでも抑制度合いを調整可能）
- 「スキ」を押した側と押された側（通知なし）、その回数と経過時間によるタイムライン上の表示優先度の増加

人を「数」で評価しない、通知に煩わされないコミュニケーションツールの社会実装を目指す（模倣はむしろ歓迎）

---

## ■ 概要

Nagi SNS（仮名）は、現代のSNSにおける以下の課題に対する解決を目指します。

- 誹謗中傷や人格攻撃が発生しやすい構造
- 数字による評価や比較の圧力
- 安心して発言できない環境

本プロジェクトは、

> **攻撃的な言動が起こりにくく、またメリットを得にくい構造**

を設計することで、心理的安全性の高い空間の実現を目指します。

---

## ■ 設計思想（最重要）

### 1. 人を数で評価しない
いいね数・フォロワー数などの可視化された評価指標に依存しない。

### 2. 攻撃行動の構造的抑制
暴言や差別を「罰する」のではなく、
それが拡散されず、メリットを持たない設計を行う。

### 3. 心理的安全性の確保
ユーザーが安心して発言できる環境を最優先とする。

### 4. 信頼の維持
一貫した設計と運用により、長期的な信頼を築く。

---

## ■ 技術スタンス

- 最小構成から開始し、継続的に改善する
- 技術は手段であり、設計思想を優先する
- **E2E（ブラウザ経由の回帰テスト）には Playwright を使う**（手順は `docs/PLAYWRIGHT_AND_TIMELINE_VERIFICATION.md`。**アルゴリズムの数式**は `docs/dev/IMPLEMENTATION_REFERENCE.md`）

- 最小構成の投稿機能を実装済み
- Google ログイン（Supabase Auth）に対応
- 初回ログイン時のニックネーム設定に対応
- ローカルおよびWebで動作確認可能
- UI/UXは未整備

---

## ■ やらないと決めたこと（現時点）

- いいね数・フォロワー数など、比較を強める数値指標の表示
- 投稿主アカウント単位の「信頼度スコア」の保持・減点運用
- 広告モデルを前提にした設計（インプレッション最適化）
- 「ユーザーを罰する」こと自体を目的にしたモデレーション設計

---

## ■ 現在の実装方針（暫定）

- いいねは関係性データとして記録するが、件数は表示しない
- 攻撃的な投稿は、将来的に投稿単位で判定し表示優先度を下げる方向で検討
- 問題判定は投稿主に理由を返し、異議申し立ての余地を残す
- 仕様は固定せず、実装と検証を通じて段階的に更新する
