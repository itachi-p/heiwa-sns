# プロジェクト憲法（heiwa-sns）

**数値・アルゴリズムの正**は **`docs/dev/IMPLEMENTATION_REFERENCE.md`** とコード。  
このファイルは **人間と Cursor の共通の境界**（やり方・止める線）を書く。

---

## 最優先（デフォルト）

- **「コードを書き換えて」「実装して」と明示されていない限り、挙動・閾値・依存関係を変える変更をしない。** 相談・経緯・未確定案の整理は、**このファイルやチャットの文**で足りることが多い。
- **未確定の案**（例: 閾値の上限、0.9 以上の扱いなど）を**勝手に実装に落とさない**。別の生成 AI で仕様を固め、**確定後の短い指示**が出たときだけコードに反映する。
- **依頼と無関係なファイルを開かない・変えない。** 説明用に**コード内コメントを増やさない**（必要なら会話かドキュメントに書く）。

## 追記のしかた（履歴を追えるように）

- 方針の転換・再発・「ここから守る」という合意ができたら、**`## YYYY-MM-DD` 見出し**で**最下部のログ**に追記する。長くなったら要約して上へ移すか、`docs/DECISIONS.md` に移す。
- **Git のコミット履歴**も、いつ何が変わったかの別の証跡になる。
