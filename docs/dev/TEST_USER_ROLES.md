# テストユーザ番号の役割（先行体験期間の運用）

**目的:** 手動検証・E2E・招待貸与で **同じ番号を食い違いなく使う**。将来の自分と Agent がここを正とする。

| 番号 | 用途 | 操作 |
|------|------|------|
| **01** | 運営本人の手動用（Gmail 等）に近い「自分用」の枠の一つ | 手動。E2E で **3 ユーザ目が必要なときだけ** E2E に割り当て可。 |
| **02** | 手動用の予備枠 | 手動。E2E で **2 ユーザ目が必要なとき**に E2E に割り当て可。 |
| **03** | **E2E 専用**（貸与アカウント想定・`E2E_LOGIN_EMAIL` 等） | **手動ログインしない。** 成否に関わらず、**繰り返しテストで引っかかる DB 状態は teardown で初期寄せ**する（ニックネームなど影響の少ない列は変えたままでよい）。実装・手順の詳細は `e2e/reset-lent-invite-user.ts` と `docs/PLAYWRIGHT_AND_TIMELINE_VERIFICATION.md`。 |
| **04 以降** | **初期体験・招待貸与のスタンバイ**（未貸与のまま同じ初期状態で揃える） | **手動 E2E には使わない。** 運営が新規登録で作るにせよ **同じ初期形に揃え**、**実ユーザーが触るまで変えない**。不要になった行は Table Editor か SQL で削除（`posts.user_id` に FK が無いため、`auth.users` を消す前にそのユーザの投稿・返信・スキ等を掃除すること）。 |

**方針変更時:** Agent は本文と異なる割り当て（例: 04 を E2E に回す）をコードに入れる前に、**ユーザーに一言確認**する。

**ローカル開発:** 左下の Next 開発インジケータは **`next.config.ts` の `devIndicators: false`** で非表示（ナビ・スクショの邪魔軽減）。
